Helm installs things. It pushes config into a cluster and walks away. If a Deployment gets deleted, its replica count changes, or a pod enters a crash loop — Helm has no idea. You have to run it again. **You are the reconciliation loop.**

An Operator eliminates that. An Operator is a Go program that runs *inside* your cluster, watches a Custom Resource you define, and continuously enforces that reality matches desired state. Not once on deploy. Continuously. Every time something changes, the controller's `Reconcile` function fires.

This post builds a real Operator with `operator-sdk`: CRD, controller, drift detection, self-healing — all on a local k3d cluster.

## Architecture

### Without an Operator

```diagram
Developer
   │
   │  helm install (manual trigger)
   ▼
Kubernetes API ──► Deployment / Service / ConfigMap
                             │
                             │  drift happens — no feedback loop
                             ▼
                    ⚠ Invisible until next helm run
```

Helm is a deployment tool, not a control plane. It renders templates, applies them, and stops. Drift is your problem.

### With an Operator

```diagram
WebApp CR                    Kubernetes API Server
spec.replicas: 2    ──────►        │
spec.image: nginx:alpine           │ watch event
                                   ▼
                          WebApp Controller (in-cluster)
                                   │
                              Reconcile()
                            ┌──────┴───────┐
                            │              │
                 Deployment missing?   Spec drifted?
                            │              │
                     Create Deployment   Patch Deployment
                     + set ownerRef           │
                            └──────┬──────────┘
                                   ▼
                          ✔ actual = desired
```

**Three core concepts:**

- **CRD** — Teaches Kubernetes your new resource type. After `kubectl apply -f crd.yaml`, `kubectl get webapps` is valid.
- **Custom Resource** — An instance of the CRD. Plain YAML with your domain's `apiVersion`/`kind`. The desired state input.
- **Controller** — A Go Pod. Watches CR objects. On every create/update/delete: runs `Reconcile`, compares desired vs actual, converges. Owner references cascade-delete child resources when the CR is deleted.

<Step number="2" title="Scaffold the operator project" time="~2 min">

```bash
mkdir -p /tmp/ep02-webapp-operator && cd /tmp/ep02-webapp-operator

operator-sdk init \
  --domain tinkerwithtech.dev \
  --repo github.com/chruzie/webapp-operator \
  --plugins go/v4 \
  --skip-go-version-check

operator-sdk create api \
  --group apps \
  --version v1alpha1 \
  --kind WebApp \
  --resource \
  --controller
```

<Callout type="info">
`--skip-go-version-check` is required with Go 1.22.x and operator-sdk v1.42+. You may see the Go toolchain auto-selector downloading build tools — this is normal, not an error.
</Callout>

<Callout type="warning">
kustomize derives the controller namespace from your directory name. `/tmp/ep02-webapp-operator` → `ep02-webapp-operator-system`. If you use a different directory, substitute that namespace in Steps 5 and 6.
</Callout>

This generates:
- `api/v1alpha1/webapp_types.go` — CRD spec struct
- `internal/controller/webapp_controller.go` — the reconciler

</Step>

<Step number="3" title="Define the WebAppSpec" time="~1 min">

Replace the spec structs in `api/v1alpha1/webapp_types.go`:

```go
type WebAppSpec struct {
    // +kubebuilder:validation:Minimum=1
    // +kubebuilder:validation:Maximum=10
    // +kubebuilder:default=1
    Replicas int32  `json:"replicas,omitempty"`
    // +kubebuilder:default="nginx:alpine"
    Image    string `json:"image,omitempty"`
}

type WebAppStatus struct {
    AvailableReplicas int32              `json:"availableReplicas,omitempty"`
    Conditions        []metav1.Condition `json:"conditions,omitempty" patchStrategy:"merge" patchMergeKey:"type"`
}
```

<Callout type="tip">
The `+kubebuilder:` comments become JSON Schema validators in the generated CRD. Kubernetes rejects any WebApp with `replicas > 10` automatically — no validation code required.
</Callout>

</Step>

<Step number="4" title="Write the reconciler" time="~3 min">

Replace the entire contents of `internal/controller/webapp_controller.go` with:

```go
package controller

import (
    "context"

    appsv1 "k8s.io/api/apps/v1"
    corev1 "k8s.io/api/core/v1"
    "k8s.io/apimachinery/pkg/api/errors"
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
    "k8s.io/apimachinery/pkg/runtime"
    "k8s.io/apimachinery/pkg/types"
    ctrl "sigs.k8s.io/controller-runtime"
    "sigs.k8s.io/controller-runtime/pkg/client"
    "sigs.k8s.io/controller-runtime/pkg/log"

    appsv1alpha1 "github.com/chruzie/webapp-operator/api/v1alpha1"
)

type WebAppReconciler struct {
    client.Client
    Scheme *runtime.Scheme
}

// +kubebuilder:rbac:groups=apps.tinkerwithtech.dev,resources=webapps,verbs=get;list;watch;create;update;patch;delete
// +kubebuilder:rbac:groups=apps.tinkerwithtech.dev,resources=webapps/status,verbs=get;update;patch
// +kubebuilder:rbac:groups=apps.tinkerwithtech.dev,resources=webapps/finalizers,verbs=update
// +kubebuilder:rbac:groups=apps,resources=deployments,verbs=get;list;watch;create;update;patch;delete

func (r *WebAppReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
    logger := log.FromContext(ctx)

    webapp := &appsv1alpha1.WebApp{}
    if err := r.Get(ctx, req.NamespacedName, webapp); err != nil {
        return ctrl.Result{}, client.IgnoreNotFound(err)
    }

    logger.Info("Reconciling WebApp", "name", webapp.Name,
        "replicas", webapp.Spec.Replicas, "image", webapp.Spec.Image)

    deploy := &appsv1.Deployment{}
    err := r.Get(ctx, types.NamespacedName{
        Name: webapp.Name + "-deployment", Namespace: webapp.Namespace,
    }, deploy)

    if errors.IsNotFound(err) {
        logger.Info("Creating Deployment for WebApp", "deployment", webapp.Name+"-deployment")
        newDeploy := r.deploymentForWebApp(webapp)
        ctrl.SetControllerReference(webapp, newDeploy, r.Scheme)
        if err := r.Create(ctx, newDeploy); err != nil {
            return ctrl.Result{}, err
        }
        logger.Info("Deployment created successfully")
        return ctrl.Result{Requeue: true}, nil
    }
    if err != nil {
        return ctrl.Result{}, err
    }

    // Drift detection: actual vs desired
    if *deploy.Spec.Replicas != webapp.Spec.Replicas ||
        deploy.Spec.Template.Spec.Containers[0].Image != webapp.Spec.Image {
        logger.Info("Drift detected — reconciling")
        deploy.Spec.Replicas = &webapp.Spec.Replicas
        deploy.Spec.Template.Spec.Containers[0].Image = webapp.Spec.Image
        if err := r.Update(ctx, deploy); err != nil {
            return ctrl.Result{}, err
        }
    }

    webapp.Status.AvailableReplicas = deploy.Status.AvailableReplicas
    return ctrl.Result{}, r.Status().Update(ctx, webapp)
}

func (r *WebAppReconciler) deploymentForWebApp(webapp *appsv1alpha1.WebApp) *appsv1.Deployment {
    labels := map[string]string{"app": webapp.Name}
    replicas := webapp.Spec.Replicas

    return &appsv1.Deployment{
        ObjectMeta: metav1.ObjectMeta{
            Name:      webapp.Name + "-deployment",
            Namespace: webapp.Namespace,
            Labels:    labels,
        },
        Spec: appsv1.DeploymentSpec{
            Replicas: &replicas,
            Selector: &metav1.LabelSelector{MatchLabels: labels},
            Template: corev1.PodTemplateSpec{
                ObjectMeta: metav1.ObjectMeta{Labels: labels},
                Spec: corev1.PodSpec{
                    Containers: []corev1.Container{{
                        Name:  "webapp",
                        Image: webapp.Spec.Image,
                        Ports: []corev1.ContainerPort{{ContainerPort: 80}},
                    }},
                },
            },
        },
    }
}

func (r *WebAppReconciler) SetupWithManager(mgr ctrl.Manager) error {
    return ctrl.NewControllerManagedBy(mgr).
        For(&appsv1alpha1.WebApp{}).
        Owns(&appsv1.Deployment{}).
        Named("webapp").
        Complete(r)
}
```

Three things to notice:

- **`deploymentForWebApp`** builds the Deployment object with the CR's spec. It's a pure constructor — no API calls.
- **`ctrl.SetControllerReference`** sets the owner reference so Kubernetes garbage-collects the Deployment when the WebApp CR is deleted.
- **`Owns(&appsv1.Deployment{})`** in `SetupWithManager` tells the controller to watch Deployments it owns — so deleting a Deployment triggers re-reconciliation automatically.

</Step>

<Step number="5" title="Build and load the image" time="~3 min">

```bash
cd /tmp/ep02-webapp-operator

make manifests generate   # Generate CRD YAML + RBAC
make build                # Compile the binary
make docker-build IMG="webapp-operator:dev"
k3d image import webapp-operator:dev -c operators
```

<Callout type="warning">
Use a non-`:latest` tag. With `:latest`, Kubernetes sets `imagePullPolicy: Always` and tries to pull from Docker Hub — which fails for a local image. Any other tag defaults to `IfNotPresent`.
</Callout>

</Step>

<Step number="6" title="Deploy the operator and apply a CR" time="~2 min">

```bash
kubectl config use-context k3d-operators

make install                            # Install the CRD
make deploy IMG="webapp-operator:dev"   # Deploy controller + RBAC

kubectl rollout status \
  deployment/ep02-webapp-operator-controller-manager \
  -n ep02-webapp-operator-system --context k3d-operators
```

<Callout type="warning">
`make install` and `make deploy` apply to whichever cluster is your current kubectl context. If you have other clusters active (e.g. from a previous episode), run `kubectl config use-context k3d-operators` first. The `k3d cluster create` command adds the context but may not set it as current if your kubeconfig wasn't previously merged.
</Callout>

```bash
kubectl create namespace webapp-demo --context k3d-operators

kubectl apply --context k3d-operators -f - <<EOF
apiVersion: apps.tinkerwithtech.dev/v1alpha1
kind: WebApp
metadata:
  name: my-webapp
  namespace: webapp-demo
spec:
  replicas: 2
  image: "nginx:alpine"
EOF
```

</Step>

## What You Built

```
k3d cluster: operators
├── ep02-webapp-operator-system/
│   └── controller-manager
│       └── WebApp reconciler → watches webapps.apps.tinkerwithtech.dev
└── webapp-demo/
    ├── WebApp/my-webapp  (desired: 2 replicas, nginx:alpine)
    └── Deployment/my-webapp-deployment  (owned by WebApp via ownerRef)
        ├── Pod/my-webapp-deployment-xxx-yyy
        └── Pod/my-webapp-deployment-xxx-zzz
```

The loop enforces desired state continuously — delete the Deployment and it's back in ~2s; delete the WebApp CR and the Deployment cascades away. Database operators, certificate operators, GitOps operators — all variations of this same structure.

Here's the silent tax every observability stack accumulates.

Prometheus client library in the app. Jaeger agent sidecar. Loki log shipper. Three SDKs to maintain, three backends to configure, three dashboards to check — and no way to correlate a trace with a metric spike unless you've wired up exemplars by hand.

**OpenTelemetry eliminates this.** One SDK. One wire protocol (OTLP). One Collector that receives all three signals and routes them to whatever backends you already use. Your Prometheus, Grafana, and Jaeger setups stay unchanged — you just stop maintaining three separate instrumentation libraries.

This post walks through a complete homelab deployment: OTel Collector with a multi-signal pipeline, Prometheus with remote-write receiver, Grafana, and a sample app emitting real OTLP data — all on k3d.

## Architecture

```diagram
Your App (any language)
   │  OTLP gRPC :4317 or HTTP :4318
   ▼
┌─────────────────────────────────────────┐
│              OTel Collector             │
│                                         │
│  Receivers        otlp, prometheus      │
│       │                                 │
│       ▼                                 │
│  Processors       memory_limiter, batch │
│       │                                 │
│       ▼                                 │
│  Exporters        debug, prometheusrw   │
└──────────────────────┬──────────────────┘
                       │
          ┌────────────┴────────────┐
          │ remote_write            │ debug logs
          ▼                         ▼
      Prometheus            Collector stdout
          │                  (traces + logs)
          ▼
        Grafana
```

The Collector is the **central bus**. It decouples your application instrumentation from your observability backends. Swap Jaeger in for traces, Loki for logs — without changing a single line of application code.

**Why the Collector matters:** without it, your app speaks Prometheus format directly, or runs a Jaeger agent sidecar, or a Fluentd log shipper — each with its own config format, failure mode, and upgrade path. With OTel, your app speaks OTLP to the Collector. The Collector translates. Backends evolve independently.

## Step-by-step

<Step number="1" title="Create the k3d cluster" time="~1 min">

```bash
k3d cluster create otel --agents 1 --wait \
  --k3s-arg "--disable=traefik@server:0"
kubectl config rename-context k3d-otel k3d-otel 2>/dev/null || true
```

<Callout type="check">
`kubectl cluster-info --context k3d-otel` returns without error.
</Callout>

</Step>

<Step number="2" title="Create namespaces" time="~30 sec">

```bash
kubectl create namespace observability --context k3d-otel
kubectl create namespace sample-app --context k3d-otel
```

</Step>

<Step number="3" title="Install kube-prometheus-stack" time="~5 min">

```bash
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo update

helm install kube-prometheus-stack prometheus-community/kube-prometheus-stack \
  --namespace observability \
  --kube-context k3d-otel \
  --set grafana.adminPassword="tinkerwithtech" \
  --set prometheus.prometheusSpec.scrapeInterval="15s" \
  --set prometheus.prometheusSpec.evaluationInterval="15s" \
  --set prometheus.prometheusSpec.enableRemoteWriteReceiver=true \
  --set alertmanager.enabled=false \
  --set nodeExporter.enabled=false \
  --set kubeStateMetrics.enabled=false \
  --wait --timeout=300s
```

<Callout type="warning">
`enableRemoteWriteReceiver=true` is required. Without it, the OTel Collector's `prometheusremotewrite` exporter fails with a 404 — Prometheus rejects remote write requests by default.
</Callout>

```bash
kubectl rollout status deployment/kube-prometheus-stack-grafana -n observability --context k3d-otel
```

</Step>

<Step number="4" title="Install the OTel Collector" time="~3 min">

```bash
helm repo add open-telemetry https://open-telemetry.github.io/opentelemetry-helm-charts
helm repo update

helm install otel-collector open-telemetry/opentelemetry-collector \
  --namespace observability \
  --kube-context k3d-otel \
  --set mode=deployment \
  --set fullnameOverride=otel-collector \
  --set image.repository=otel/opentelemetry-collector-contrib \
  --values - <<'HELM_VALUES'
config:
  receivers:
    otlp:
      protocols:
        grpc:
          endpoint: 0.0.0.0:4317
        http:
          endpoint: 0.0.0.0:4318
    prometheus:
      config:
        scrape_configs:
          - job_name: "otel-collector-self"
            scrape_interval: 10s
            static_configs:
              - targets: ["localhost:8888"]

  processors:
    batch:
      timeout: 5s
      send_batch_size: 512
    memory_limiter:
      check_interval: 1s
      limit_mib: 250

  exporters:
    debug:
      verbosity: detailed
    prometheusremotewrite:
      endpoint: "http://kube-prometheus-stack-prometheus.observability.svc.cluster.local:9090/api/v1/write"
      tls:
        insecure: true

  service:
    pipelines:
      traces:
        receivers: [otlp]
        processors: [memory_limiter, batch]
        exporters: [debug]
      metrics:
        receivers: [otlp, prometheus]
        processors: [memory_limiter, batch]
        exporters: [debug, prometheusremotewrite]
      logs:
        receivers: [otlp]
        processors: [memory_limiter, batch]
        exporters: [debug]

ports:
  otlp:
    enabled: true
    containerPort: 4317
    servicePort: 4317
    protocol: TCP
  otlp-http:
    enabled: true
    containerPort: 4318
    servicePort: 4318
    protocol: TCP
  metrics:
    enabled: true
    containerPort: 8888
    servicePort: 8888
    protocol: TCP
HELM_VALUES
```

<Callout type="info">
`image.repository=otel/opentelemetry-collector-contrib` is required from chart v0.110+. The contrib distribution includes the `prometheusremotewrite` exporter. The core distribution does not.
</Callout>

```bash
kubectl rollout status deployment/otel-collector -n observability --context k3d-otel
```

</Step>

<Step number="5" title="Deploy the sample app" time="~2 min">

A minimal load generator that sends all three OTLP signals every 10 seconds — no language SDK required:

```bash
kubectl apply --context k3d-otel -f - <<'EOF'
apiVersion: apps/v1
kind: Deployment
metadata:
  name: sample-app
  namespace: sample-app
spec:
  replicas: 1
  selector:
    matchLabels:
      app: sample-app
  template:
    metadata:
      labels:
        app: sample-app
    spec:
      containers:
        - name: load-generator
          image: curlimages/curl:latest
          command:
            - /bin/sh
            - -c
            - |
              COUNT=0
              while true; do
                COUNT=$((COUNT + 1))
                NOW_NS=$(date +%s)000000000
                START_NS=$(( $(date +%s) - 1 ))000000000
                # Traces
                curl -sf -X POST http://otel-collector.observability.svc.cluster.local:4318/v1/traces \
                  -H "Content-Type: application/json" \
                  -d "{\"resourceSpans\":[{\"resource\":{\"attributes\":[{\"key\":\"service.name\",\"value\":{\"stringValue\":\"sample-app\"}}]},\"scopeSpans\":[{\"spans\":[{\"traceId\":\"0102030405060708090a0b0c0d0e0f10\",\"spanId\":\"010203040506070${COUNT}\",\"name\":\"GET /api/users\",\"kind\":2,\"startTimeUnixNano\":\"${START_NS}\",\"endTimeUnixNano\":\"${NOW_NS}\",\"status\":{\"code\":1}}]}]}]}" || true
                # Metrics
                curl -sf -X POST http://otel-collector.observability.svc.cluster.local:4318/v1/metrics \
                  -H "Content-Type: application/json" \
                  -d "{\"resourceMetrics\":[{\"resource\":{\"attributes\":[{\"key\":\"service.name\",\"value\":{\"stringValue\":\"sample-app\"}}]},\"scopeMetrics\":[{\"metrics\":[{\"name\":\"http_requests_total\",\"sum\":{\"dataPoints\":[{\"startTimeUnixNano\":\"${START_NS}\",\"timeUnixNano\":\"${NOW_NS}\",\"asInt\":${COUNT}}],\"aggregationTemporality\":2,\"isMonotonic\":true}}]}]}]}" || true
                # Logs
                curl -sf -X POST http://otel-collector.observability.svc.cluster.local:4318/v1/logs \
                  -H "Content-Type: application/json" \
                  -d "{\"resourceLogs\":[{\"resource\":{\"attributes\":[{\"key\":\"service.name\",\"value\":{\"stringValue\":\"sample-app\"}}]},\"scopeLogs\":[{\"logRecords\":[{\"timeUnixNano\":\"${NOW_NS}\",\"severityText\":\"INFO\",\"body\":{\"stringValue\":\"handled GET /api/users\"}}]}]}]}" || true
                sleep 10
              done
EOF

kubectl rollout status deployment/sample-app -n sample-app --context k3d-otel
```

</Step>

<Step number="6" title="Start port-forwards" time="~30 sec">

```bash
kubectl port-forward -n observability svc/kube-prometheus-stack-grafana 3000:80 \
  --context k3d-otel &
kubectl port-forward -n observability svc/kube-prometheus-stack-prometheus 9090:9090 \
  --context k3d-otel &
kubectl port-forward -n observability svc/otel-collector 4318:4318 \
  --context k3d-otel &
```

<Callout type="info">
The `&` runs each port-forward in the background of your current shell. They terminate when you close the terminal — re-run in a new session if needed.
</Callout>

| UI | URL | Credentials |
|----|-----|-------------|
| Grafana | http://localhost:3000 | admin / tinkerwithtech |
| Prometheus | http://localhost:9090 | — |

</Step>

## What You Built

```
k3d cluster: otel
└── observability/
    ├── otel-collector     ← Central signal bus (OTLP in → debug + promremote out)
    ├── prometheus         ← Stores metrics (remote-write receiver enabled)
    └── grafana            ← Visualises Prometheus metrics

sample-app/
└── sample-app             ← curl container; sends OTLP traces + metrics + logs every 10s
```

**Key facts to remember:**

| Concept | Detail |
|---------|--------|
| OTLP | Standard wire protocol for all three signals — swap any backend without touching app code |
| OTel Collector | Stateless, scalable — run as DaemonSet for logs, Deployment for traces/metrics |
| `collector-contrib` image | Required for `prometheusremotewrite` exporter — core image lacks it |
| Remote-write receiver | Must enable on Prometheus: `--web.enable-remote-write-receiver` or the Helm flag above |
| CNCF momentum | #2 most active CNCF project — SDK support spans 11+ languages |

## Teardown

When you're done experimenting, clean up everything:

```bash
# Stop any port-forwards
pkill -f "port-forward.*prometheus" 2>/dev/null
pkill -f "port-forward.*grafana" 2>/dev/null

# Delete the cluster
k3d cluster delete otel
```
